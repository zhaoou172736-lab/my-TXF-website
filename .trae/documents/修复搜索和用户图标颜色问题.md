# 问题分析
从截图可以看到，搜索和用户图标在滚动后变成了黑色方块，而不是像logo那样保持白色图标。这是因为：
1. Lucide图标组件的颜色是通过`stroke`属性控制的，而不是通过CSS的`color`属性
2. 当页面滚动时，`finalTextColor`从`text-white`变成`text-[#181818]`，直接应用到图标组件上导致显示异常
3. Logo使用直接SVG元素，通过`fill="currentColor"`正确继承父元素颜色，而Lucide图标组件没有使用这种方式

# 解决方案
1. **修改Lucide图标实现**：将Lucide图标组件替换为直接的SVG元素，像logo一样使用`currentColor`继承父元素颜色
2. **统一颜色控制**：确保所有图标都通过`currentColor`继承父元素的颜色，这样在滚动时能正确变色
3. **保持一致的视觉效果**：确保图标在滚动前后都能清晰显示，没有黑色边框

# 实现步骤
1. 替换搜索和用户图标为直接SVG元素
2. 设置SVG的`fill`和`stroke`属性为`currentColor`
3. 确保图标能正确继承父元素的颜色，在滚动时平滑过渡
4. 验证修改后的效果

# 预期结果
- 搜索和用户图标在页面顶部时显示为白色
- 当页面滚动后，图标颜色随导航栏一起变为黑色
- 图标始终清晰可见，没有黑色边框
- 与logo的颜色变化保持一致
